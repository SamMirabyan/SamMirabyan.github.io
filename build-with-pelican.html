Build a static site with Pelican and GutHub Pages the easy way

Posted by Sam Mirabyan on Sun 05 June 2022 Updated on Wed 08 June 2022

Hi, folks!

Today I’m going to share my experience of using Python’s Pelican library for my personal project.

A little backstory

As a beginner programmer, I needed a way to communicate my thoughts and my skills to recruiters and hiring managers. That’s why I decided to create a blog-portfolio ish website and put it out there in the web. Considering that I’m fresh out of Python bootcamp and that my main focus (competence) is building REST APIs, the first though that pumped in my head was: build a Django app, attach some front face (templates or whatever) and I’m good to go.

So the next part is to make the site visible to world. Well, what free options do we have? I guess the main option is to get a free domain name from noip.com and host your website on Heroku. You can serve one app constrained to 500 MB of RAM consumption. And it’s perfect for a single little app like mine, because other way I would need to a buy a VPS (which is a fair amount of money) or configure my own server (from an old laptop for example). After few days of reflection, I dropped this idea, mostly because I found the idea of using Django for a static web application somewhat like an overkill.

That’s how I came to a decision to host a static website on GitHub Pages. It is free, it has a concise domain name (‘username.gihub.io’ instead of ‘kissmyhumbleunicorn1984.com.hopto.org.fun.’) and it’s fairly simple to manage your static content (posts, project summary and etc.). By default GitHub Pages uses Jekyll as it’s static site generator. It is nice, but as a Python fan, I wanted something written in Python to be able to tweak the source code if needed. So I found a very interesting tool called Pelican.

What is Pelican?

In Python world, Pelican is a static site generator (SSG). It lets you generate html docs coupled with CSS and JS using declarative syntax. The basic principle is simple: you write some content to a markdown file (or RST file, but I will stick to markdown), add some metadata (author, publish date, tag etc.). Then you run some pelican instructions and it creates a fully functional html files. After that, you can host these files practically anywhere on the web.

My setup

OS: Linux Mint 20.1 (same as Ubuntu) Python: 3.8.10 version_control: git 2.36.1 text editor: vscode 1.67.2

Getting started with Pelican

The process of setting everything up is pretty simple.

Create the root directory to host your future site

mkdir pelican_site

Inside this directory create a python virtual environment in order not to mess local dependencies with your system.

python3 -m venv venv

Pay attention that second venv is the name of the virtual env folder. It can be whatever the name you want.

Inside root directory of the project create another one to store custom themes for pelican

mkdir themes

Activate virtual environment

source venv/bin/activate

Install Pelican and Markdown packages

pip install Pelican Markdown

Some folks recommend installing all dependencies from a requirements.txt file generated from pip freeze command. I personally recommend not doing so. Pip freeze extracts all dependencies installed into your virtual environment so you end up with a rather verbose output. Like this

Well, we’ve installed everything we need, let’s set up our website

Create a directory to hold the source files

mkdir source

Navigate to this directory

cd source/

Run this command to scaffold your future website’s structure

pelican-quickstart

Then answer several questions

> Where do you want to create your new web site? [.] # in curren directory
> What will be the title of this web site? my-portfolio-site
> Who will be the author of this web site? Sam Mirabyn
> What will be the default language of this web site? [en] # press enter
> Do you want to specify a URL prefix? e.g., https://example.com   (Y/n) n # will setup this later
> Do you want to enable article pagination? (Y/n) y 
> How many articles per page do you want? [10] 6
> What is your time zone? [Europe/Rome] Europe/Moscow
> Do you want to generate a tasks.py/Makefile to automate generation and publishing? (Y/n) y
> Do you want to upload your website using FTP? (y/N) n
> Do you want to upload your website using SSH? (y/N) n
> Do you want to upload your website using Dropbox? (y/N) n
> Do you want to upload your website using S3? (y/N) n
> Do you want to upload your website using Rackspace Cloud Files? (y/N) n
> Do you want to upload your website using GitHub Pages? (y/N) y
> Is this your personal page (username.github.io)? (y/N) y

That’s all. Pelican has generated your project structure for you. What do we have here?

# bash
tree

# output
├── content
├── Makefile
├── output
├── pelicanconf.py
├── publishconf.py
└── tasks.py
  • Makefile and tasks.py are two files to automate generation and publishing of your site. They offer some usefull functionality, but I won't be using them in this tutorial. We will build everything by hand to get the feel of how things work.
  • pelicanconf.py and publishconf.py are two settings files.
  • content/ directory holds raw content files and output/ directory holds generated html files. At first both of them are empty.

All right, it’s time to write some content

Navigate to contents/ directory

cd contents/

In order to create a post simply create a markdown file touch first_post.md Open this file with a text editor of your choice. Fill the metadata …. Add some content … Save it and return to command line To generate html files from you content run pelican content Now look how your output folder has changed Here we can see new html file. Let’s open and inspect it a little bit. Here we have our properly designed html file with styles and scripts.

Now you may wonder what your files look like. Pelican provides this functionality as well. pelican - - listen This command starts a Python http server and serves your html pages on it. Open your browser and type into address bar localhost:8000 Now you should see properly designed html page. And the process is the same for all of the other articles. Create a markdown file, write some content to it, save it, run pelican content, run pelican - - listen to check the result.

Next let’s add some common pages like ‘About’ or ‘Contact’ to your website. Pelican makes a distinction between articles and pages. -Articles are like posts, you may have lots of them. -Pages store some common information and change rarely. For now I need two pages: About and Contact. About will have few words about my background, skills, and plans. Contact will store my GitHub, Telegram and other handles. cd to contents/ directory and create pages/ directory inside it. You may also want to create articles/ directory to keep your articles in one place and maintain the structure of the project consistent. Then cd to pages/ directory and create a file about.md Open this file and write some useful information about what a great person you are. Then stop the server if you haven’t yet with ctrl+c combination. run pelican content and pelican - -listen And refresh your browser’s localhost page. There should appear your ‘About’ page. The process of adding ‘Contact’ page is the same so you can do it yourself.

Now let’s change the look and feel of the website. The greatest thing here is that you don’t need to create your own templates and mess with html, CSS and other visual thing (unless this is what you want to do), Pelican’s got you covered. Firstly list all preinstalled themes with pelican-themes - -list command. Next visit pelican-themes website and choose a theme of your liking. One popular theme is Flex, but for my project I chose … After you chose the preferable theme head to github.com/getpeliacn/pelican-themes, find chosen theme and clone it to themes directory. Now install your new theme with command pelican-themes - -install - - verbose (to see the output) After this run the content command and start the dev server. The site theme should change.

All right we have a website that works on your local machine, let’s pair it with GitHub Pages. I will keep the process as simple as it can be in order to have the feeling of control over the process. Navigate to your GitHub account, create new repository and name it .github.io (mine is ‘sammirabyan.gitgub.io’), press create button. After that you will be redirected to the default page of starter options. Copy the link, switch to bash and navigate to output/ directory. Here you need to run some instructions: git init git checkout –b main git add . git commit –m ‘initial commit’ git remote add origin … git push –U main origin Great, we have sent our static files to our github pages repository, you can switch to repository from your browser and notice one orange mark <> GitHub by default runs Actions workflow to ensure everything is set up properly. After it finished you can navigate to .github.io and see your site running. Looks nice!

Next what I recommend to do is to push your source files to GitHub as well. It will save you a lot of time in case of file loss on your local machine or in case you need to work with you site using another set up. Create new GitHub repository and name it something like ‘pelican_site_source_code’ or whatever the name you want. Then we need to do the trick like we did with our github.io repository but with a little more effort. git init git checkout –b main touch ‘.gitiignore’ Add this code to .gitignore You may also want to add a README.md file

Now we have our source files safely stored in a remote repository. Pay attention that in this case you need to maintain (manually) the state of two repositories – one for output and another for source code. There are some tools to make life easier and less error prone, but for now
